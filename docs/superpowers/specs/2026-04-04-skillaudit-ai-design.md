# Design Spec: SkillAuditAI (Security Auditor for AI Skills)

**Date:** 2026-04-04  
**Status:** Draft  
**Target:** Node.js (TypeScript) CLI & Core Library  

---

## 1. Overview
SkillAuditAI is a security-focused scorecard tool for AI skills defined in the `SKILL.md` format (as specified by [agentskills.io](https://agentskills.io/specification)). Inspired by the OSSF Scorecard, it evaluates skills across multiple safety parameters to provide a unified "Safety Score" (0-10).

## 2. Goals & Success Criteria
- **Security First:** Identify dangerous tools, over-permissioning, and prompt-level risks.
- **Transparency:** Provide clear justifications for every score.
- **Extensibility:** Allow new audit checks to be added as plugins.
- **SaaS-Ready:** Decouple the engine from the CLI to support future web-based integration.

## 3. Architecture

### 3.1 Components
- **`@skillauditai/core`**: The stateless engine. It handles parsing, check orchestration, and report generation.
- **`@skillauditai/cli`**: The user-facing tool that manages file I/O, configuration, and terminal rendering.

### 3.2 The Audit Engine Pipeline
1. **Load/Parse:** Reads `SKILL.md` and converts it into a structured `SkillAST`.
2. **Context Enrichment:** (Optional) Fetches external metadata (e.g., GitHub API for reputation).
3. **Check Execution:** Runs a registry of `BaseCheck` implementations in parallel.
4. **Scoring & Weighting:** Aggregates individual check scores into a weighted final scorecard.
5. **Report Generation:** Produces an `AuditReport` (JSON/SARIF/Table).

## 4. Audit Check Categories

### 4.1 Static Analysis (Internal Safety)
| Check ID | Description | Weight |
|----------|-------------|--------|
| `dangerous-tools` | Flags risky tools like `run_shell_command` or filesystem writes. | Critical (1.0) |
| `least-privilege` | Compares requested tools against the skill's stated intent. | High (0.8) |
| `prompt-injection` | Detects lack of delimiters or missing "jailbreak" protections. | High (0.8) |
| `secret-scanning` | Searches for hardcoded API keys/tokens in examples/prompts. | Critical (1.0) |

### 4.2 Reputation & Provenance (External Trust)
| Check ID | Description | Weight |
|----------|-------------|--------|
| `verified-author` | Validates author identity via Git/GitHub/Trusted Registry. | Medium (0.5) |
| `maintenance` | Checks for versioning (SemVer) and recent update history. | Low (0.3) |
| `dependency-audit` | Evaluates the safety of any imported tools/sub-skills. | Medium (0.5) |

## 5. Technical Implementation
- **Language:** TypeScript 5.x  
- **Parsing:** Custom Markdown parser targeting the `agentskills.io` spec sections (Tools, Instructions, Examples).
- **Concurrency:** Uses `Promise.allSettled` for parallelized check execution.
- **Output Formats:**
    - `table`: High-fidelity CLI view (default).
    - `json`: Machine-readable for SaaS/API.
    - `sarif`: Industry-standard for security tool interoperability.

## 6. Future SaaS Integration
- **Caching:** Every `AuditReport` will include a SHA-256 hash of the skill content.
- **Signatures:** Support for cryptographically signing reports to prove they were generated by SkillAuditAI.
- **API Mode:** The core library will be "edge-ready" (minimal dependencies) to run in Vercel/Cloudflare workers.
